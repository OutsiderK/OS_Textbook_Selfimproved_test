# ch02 补充：中断源、中断装置与中断处理流程

对应主稿位置：第 2 章「2.6 中断机制：硬件与软件如何协作」。

## 知识点要点

- 中断源分两类：外中断来自处理器之外的设备或时钟，与现行指令无关，通常在两条指令之间响应；异常由现行指令引起，在一个指令周期内处理。
- 中断硬件由 CPU 和级联的中断控制器组成：CPU 通过 INT/INTA 与中断控制器交互，主中断控制器连接时钟、键盘等 IRQ0-IRQ7，从中断控制器扩展 IRQ8 以后线路。
- 中断装置的工作过程：中断源进入中断装置 → 中断控制部件写中断寄存器 → 现行 PSW 保存到内存 → 按中断向量找到处理入口。
- 软件侧：中断向量表保存各中断服务程序入口地址；软件进一步保护未由硬件保存的现场，处理后恢复正常操作。
- 完整链条：发现中断源 → 硬件保护现场 → 转处理程序 → 软件保护现场 → 处理 → 必要时调度 → 恢复现场。

## 与原章节的连接

第 2 章在 2.2 节把中断列为操作系统重新获得控制权的三条路径之一，但当时只点到为止。这一节把那条路径展开成与系统调用并列的完整机制：硬件先用中断装置捕获、记录并保存现行 PSW，再交给软件按向量表分派处理。它解决了原章节搁置的问题——“外部事件发生时，内核究竟靠什么夺回处理器、又怎样准确跳到处理代码”。

## 自测

1. 外中断和异常在触发原因、响应位置、屏蔽与嵌套上有什么区别？
   **解答要点**：外中断由外部设备/时钟触发、与现行指令无关、在指令之间响应、可被屏蔽位屏蔽；异常由现行指令引起、在指令周期内处理、无法靠屏蔽避免。

2. 一次中断从硬件发现到软件处理通常经过哪些步骤？
   **解答要点**：发现中断源；硬件保护现场（保存现行 PSW）；按中断向量转向处理程序；软件进一步保护现场；执行处理；必要时进入调度；恢复现场并返回断点。
